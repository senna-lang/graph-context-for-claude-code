グラフ文脈展開 → Claude Code に 何がpushされるか

Obsidianで選択 → getSelectionWithContext がノートを深さ1で辿り、EnrichedContext JSON にして MCP経由でClaude Codeへ渡す。

対象: src/context/context-builder.ts · link-expander.ts · tools/context-tools.ts

①データの流れ

選択 / pushカーソル選択 or selection_changed (debounce)
→
展開 (depth-1)embed=本文 / wikilink=要約 / backlinks収集
→
サイズ上限embed 2000 / wiki 200 / total 8000 / backlinks 20
→
EnrichedContext JSONMCP tool result としてccへ

②具体例 — ソースノートと展開グラフ

# Project Atlas

## Architecture
The system relies on ![[Auth Service#Overview]] for identity.
See also [[Data Pipeline]] and [[Deployment]].

## Risks
Vendor lock-in on [[Auth Service]] remains open.
現在のノート / 選択 embed（本文展開） wikilink（要約）
展開グラフ — ノードをhoverで関係強調depth = 1
current embed wikilink backlink
展開のルール（コードより）
  • embed ![[Auth Service#Overview]] → 本文をそのまま取得。#heading指定はそのsectionだけ、^blockIdはそのブロックだけ。最大 2000字。
  • wikilink [[Data Pipeline]] → frontmatter + 最初の段落だけの要約。最大 200字。本文全部は送らない。
  • backlink このノートを参照している他ノート（path + name）。最大 20件。
  • guard 深さ1のみ・自己参照スキップ・未解決リンクは (unresolved) として明示・コードブロック内のリンクは無視。

③サイズ上限（プロンプト肥大を防ぐ安全弁）

対象定数上限超過時
embed 本文EMBED_CAP2000 字…[truncated] 付与 + truncated:true
wikilink 要約WIKILINK_SUMMARY_CAP200 字…[truncated] 付与
backlinks 件数BACKLINKS_CAP20 件切り捨て + backlinks:true
合計ペイロードTOTAL_CONTEXT_CAP8000 字expandedText を末尾から削る + totalContext:true

④実際に push される JSON ペイロード

これが getSelectionWithContext の戻り値（MCP tool result）。色付き行が展開で追加された文脈。

EnrichedContext (= state.context)JSON.stringify → ToolCallResult.content[0].text
// ── 選択そのもの (SelectionState) ──
{
  "filePath": "/vault/Notes/Project Atlas.md",
  "text": "The system relies on ![[Auth Service#Overview]] for identity.",
  "selection": { "start":{"line":9,"character":0}, "end":{"line":9,"character":59}, "isEmpty":false },

  // ── ここから下が「グラフ展開」で足される文脈 ──
  "context": {

    // 選択位置の見出しパンくず
    "headingPath": [ "# Project Atlas", "## Architecture" ],

    "frontmatter": { "status":"active", "owner":"senna", "tags":["project","infra"] },

    // embed: 本文を展開（≤2000）
    "expandedText": "## Overview\nAuth Service issues short-lived JWTs via OAuth2…",

    "linkedSummaries": [
      { "linkText":"Auth Service", "resolvedPath":"/vault/Services/Auth Service.md",
        "kind":"embed", "expandedText":"## Overview\nAuth Service issues…",
        "truncated":false, "unresolved":false },
      { "linkText":"Data Pipeline", "resolvedPath":"/vault/Notes/Data Pipeline.md",
        "kind":"wikilink", "summary":"owner: maya\nIngests events from Kafka into the lake.",
        "truncated":false, "unresolved":false },
      { "linkText":"Deployment", "resolvedPath":null,
        "kind":"wikilink", "summary":"[[Deployment]] (unresolved)",
        "truncated":false, "unresolved":true }
    ],

    // backlinks: このノートを参照している他ノート（≤20）
    "backlinks": [
      { "path":"/vault/Notes/Q3 Roadmap.md", "name":"Q3 Roadmap" },
      { "path":"/vault/Notes/Infra Review.md", "name":"Infra Review" }
    ],

    "truncated": { "expandedText":false, "totalContext":false, "backlinks":false }
  }
}
要点 — Claude Codeから見ると、選択テキストだけでなく 見出しパンくず frontmatter embed先の本文 リンク先の要約 被リンク一覧 が1つのJSONで届く。「このノート単体」では分からない周辺のグラフ文脈を、上限付きで安全に渡すのがこの機能の役割。
生成: src/context/* の実装に基づく可視化 · ノード/エッジ・色分け・JSONハイライトはコード上の型(EnrichedContext, LinkSummary)に対応。
